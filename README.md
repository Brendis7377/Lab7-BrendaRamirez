Question 1 : Where would you fit your automated tests in your Recipe project development pipeline? Select one of the following and explain why.
Options: 
1.Within a Github action that runs whenever code is pushed 
2.Manually run them locally before pushing code
3.Run them all after all development is completed
Answer : I would fit in my automated tests within a Github action that runs whenever code is pushed. I would do this because this way, we are able to catch bugs and errors through small actions as well as before they are fully pushed into a project. It allows us to manage the testing appropriately and it also makes development more efficient because every update is able to be consistently checked without relying on someone to remember to run tests manually and makes team work easier/more efficient.

Question 3 : What is the difference between navigation and snapshot mode? 
Answer : Navigation mode analyzes a webpage from the moment it loads and focuses more on overall page load performance and user experience. While napshot mode analyzes the webpage in its current state at that exact moment and is mainly used to identify accessibility issues, but it does not analyze loading performance or interactions on the webpage.
